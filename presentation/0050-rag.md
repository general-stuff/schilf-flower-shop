# RAG

## Requirements

We want to add a RAG feature to our app. The user can ask a flower-related question, we search in the `flowers` table using embeddings, and try to answer the question based on the most similar 3 flowers.

## Database

Create a table `flower_embeddings` with the following columns:

* `id` — PK (same PK as in the `flowers` table)
* `description_hash` — SHA-256 hash of the flower description
* `description_vector` — vector embedding of the flower description (BLOB)

The existing _fill_ operation on `flowers` MUST NOT fill the `flower_embeddings` table.

## User Interface

We need a new page "Flower Q&A".

* User input area on the top of the page.
* Same "send" button as in the chat page. Height must be equal to the input area height. Button must be square.
* Result of the RAG process below the input area (sanitized HTML built from the markdown returned by the RAG process)

## Data Access Library

* `getFlowersToEmbed`:
  * Read all flowers from the `flowers` table.
  * For each flower, compute the SHA-256 hash of the description.
  * Check if the flower already has an embedding in the `flower_embeddings` table (id and hash must match).
  * Return all flower IDs that do not have an up-to-date embedding.

* `storeFlowerEmbeddings`:
  * Input: flower PK, SHA-256 hash of the description, vector embedding of the description
  * Store the input in the `flower_embeddings` table (overwrite the existing embedding of the same ID or create a new row if it does not exist)

* `searchFlowers`:
  * Input: embedding of the user's question
  * Iterate over all flowers in the `flower_embeddings` table.
  * For each flower, compute the dot product using the `dot` function of `mathjs`.
  * Return the 3 flowers (PKs and descriptions) with the highest dot product.

## Search Process

* When the user loads the flower Q&A page, the page updates the embeddings in the DB using `getFlowersToEmbed` and `storeFlowerEmbeddings`.
  * While this process is running, display a "loading" message.
  * Use the AI SDK to get the embeddings. Use OpenAI's `text-embedding-3-large` model.
* When the user sends a message:
  * Calculate embeddings of the user's question.
  * Call `searchFlowers` to get the most similar flowers.
  * Use the AI SDK (same model as for the chat) to generate a response based on the most similar flowers. Design an appropriate system prompt. The prompt must advise the agent to answer only questions about the flowers.

## Technical Requirements

* When calculating the flower embeddings, calculate ALL embeddings in one request. Do NOT calculate them one by one.
* Similar to the chat page, use a streaming API endpoint (SSE) to stream the response to the user.
* Use the existing SSE parsing logic to parse the response.

## Quality Assurance

* Add an end-to-end test verifying that the flower Q&A page can be loaded and that the user can send a message to the bot.
* Add an end-to-end test verifying that the bot can answer a question about the flowers (content does not need to be checked, only that the response is returned to the user).
