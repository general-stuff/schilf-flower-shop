# Chat Bot

## Requirements

Add a chat page with a client-side chat component to our web app. For now, it will just be a static system prompt. Later, we will add functionality to place orders via the chat.

## User Interface

* Input area (2 lines with text wrapping) for the user at the bottom of the page
* To the right of the input area, a button to send the message.
  * Use a Send icon from Font Awesome.
  * Button height must be equal to the input area height. Button must be square.
* A chat history area above the input area.
* Use a scrollable div for the chat history. The input area must not be part of the scrollable div.
* User messages should be aligned to the right, bot messages to the left. Use distinct styling to clearly distinguish between user and bot messages.
* The input area must not be disabled when the chat is loading. However, the send button MUST be disabled.
* When a chat response is finished, set the focus to the input area.
* Whenever a new user message is added to the chat or the chat component receives a new message from the server, scroll to the bottom of the chat history.

## Technical Requirements

Use the Vercel AI SDK to implement the chat functionality.

The chat process must be implemented as follows:

1. Chat component calls an API endpoint. It uses the OpenAI API's **Conversation API** to create a conversation ID.
   * Later, pass the conversation ID to the AI SDK so that OpenAI manages the conversation history.
   * Do NOT store the conversation history yourself. Do NOT use the previous response ID.
   * Use `iron-session` to store the conversation ID in a cookie. The client should NEVER see the conversation ID unencrypted.
* Use the OpenAI **Response API** (not Chat Completions API) in the background.
  * Use the gpt-5.4 model. Make the model name configurable in the .env.local file.
* The .env.local file in the web app folder of the project already contains the `OPENAI_API_KEY`.
* The logic of the interaction with the AI should be part of the web app as it will only be used there (not in the console app).
* Use a streaming API endpoint in the web app to stream the token responses to the web client.
  * Use the SSE protocol
  * Use named SSE events as we will later not just transfer text responses but also data about tool calls and other events.
  * Use a POST request to send the message to the API endpoint. As a result, you cannot use `EventSource` on the client side. Add custom SSE parsing to the web app.
* Use `marked` to convert markdown response from OpenAI to sanitized HTML.
* Make sure to add CSS styling to the chat history to nicely format unordered lists, ordered lists, and tables (here it is ok to use HTML tables, no need for CSS Grid or Flexbox).
* Use a placeholder system prompt, so that the bot can answer questions about the flower shop. We will work on this later.

## Testing

* Custom SSE parsing must be thoroughly tested using unit tests.
* Add an end-to-end test verifying the existance of the input area, the send button, and the (initially empty) chat history.
* Add an end-to-end test verifying that the bot can answer a question (e.g. "Hi") with any non-empty response within max. 60 seconds.
