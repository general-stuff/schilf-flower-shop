# Chat Bot

## Requirements

Add a chat functionality to our web app. You can replace the existing demo home page with the chat interface. However, make it a component so that we can move the chat to other pages later.

## User Interface

* Input area (2 lines with text wrapping) for the user at the bottom of the page
* To the right of the input area, a button to send the message. Use a Send icon from Font Awesome.
* A chat history area above the input area. Use a scrollable div. User messages should be aligned to the right, bot messages to the left. Use distinct styling to clearly distinguish between user and bot messages.

## Technical Requirements

* Use the Vercel AI SDK to implement the chat functionality.
* Use the OpenAI SDK to create a conversation ID. Pass the conversation ID to the AI SDK so that OpenAI manages the conversation history.
* Use the OpenAI Response API (not Chat Completions API) in the background.
* the .env file in the root folder of the project already contains the `OPENAI_API_KEY`.
* Make sure to encapsulate the interaction with the AI SDK and OpenAI in the library, not in the web app or the console app.
* Use a streaming API endpoint in the web app to stream the token response to the web client.
  * Use the SSE protocol
  * Use named SSE events as we will later not just transfer text responses but also data about tool calls and other events.
* Use `marked` to convert markdown response from OpenAI to HTML.
* Make sure to add CSS styling to the chat history to nicely format unordered lists, ordered lists, and tables (here it is ok to use HTML tables, no need for CSS Grid or Flexbox).
* Put a placeholder system prompt in the library, so that the bot can answer questions about the flower shop. We will work on this later.
